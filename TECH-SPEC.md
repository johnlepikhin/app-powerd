# Техническое задание

## Сервис управления фоновыми приложениями для экономии батареи Linux

## 1. Цель проекта

Разработать пользовательский сервис (user-level daemon) для Linux, который уменьшает энергопотребление системы за счёт автоматического управления фоновыми графическими приложениями.

Сервис должен:

* отслеживать активное окно пользователя;
* выявлять приложения, потерявшие фокус;
* применять к ним политики энергосбережения;
* автоматически возвращать приложения в активное состояние при взаимодействии пользователя.

Основная задача — **уменьшить количество wakeups CPU и фоновой активности приложений**, сохраняя при этом корректную работу пользовательской среды.

Сервис должен работать **без модификации приложений**.

---

# 2. Основные функции

Сервис выполняет следующие задачи:

### 2.1 Мониторинг состояния окон

Сервис должен отслеживать:

* активное окно
* смену фокуса
* появление новых окон
* закрытие окон
* тип окна (normal / dialog / fullscreen)

Источники информации:

* X11
* Wayland (через соответствующие протоколы compositor)

Поддержка должна быть архитектурно разделена:

```
FocusBackend
 ├─ X11
 ├─ Wayland
 └─ Desktop specific (GNOME/KDE/Sway при необходимости)
```

---

### 2.2 Идентификация приложений

Каждое окно должно быть сопоставлено с процессом.

Необходимо определить:

* PID
* executable
* командную строку
* desktop file
* WM_CLASS / app_id
* дерево процессов

Цель — определить **группу процессов приложения**.

---

### 2.3 Формирование группы процессов приложения

Все действия должны выполняться не над одним PID, а над **группой процессов приложения**.

Целевая сущность — **cgroup приложения**.

Алгоритм:

1. определить root PID приложения
2. собрать все дочерние процессы
3. переместить их в отдельную cgroup (если она не существует)

---

### 2.4 Применение политики энергосбережения

Поддерживаются три режима:

### ignore

Никаких действий.

---

### throttle

Снижение приоритета выполнения.

Механизмы:

* nice
* cgroup CPUWeight
* cgroup CPUQuota

Цель:

* снизить CPU usage
* уменьшить влияние на частоту CPU

---

### freeze

Полная остановка приложения.

Механизм:

* cgroup freezer (предпочтительно)
* fallback: SIGSTOP/SIGCONT

При freeze:

* процессы полностью перестают исполняться
* wakeups CPU прекращаются

---

### 2.5 Возобновление приложения

Приложение должно быть автоматически возобновлено при:

* получении фокуса
* клике по окну
* выборе окна через Alt-Tab
* pointer enter

Resume должен происходить **до передачи фокуса**, чтобы избежать визуальной задержки.

---

### 2.6 Отложенная остановка приложения

После потери фокуса приложение не должно сразу замораживаться.

Используется таймер:

```
suspend_delay
```

Если пользователь вернул фокус раньше — остановка отменяется.

---

### 2.7 Периодическое кратковременное пробуждение

Опциональная функция.

Замороженное приложение может временно размораживаться:

```
каждые N секунд
на M секунд
```

Это нужно для:

* обработки сетевых событий
* обновления уведомлений

Функция должна быть отключена по умолчанию.

---

# 3. Ограничения безопасности (Guards)

Перед применением freeze/throttle сервис должен проверять условия.

Если условие выполняется — действие отменяется.

Поддерживаемые проверки:

### аудио

Если процесс воспроизводит звук — freeze запрещён.

### микрофон

Если приложение использует микрофон — freeze запрещён.

### камера

Если используется камера — freeze запрещён.

### fullscreen

Если окно fullscreen — freeze запрещён.

### недавний пользовательский ввод

Если пользователь взаимодействовал с системой недавно — freeze откладывается.

# 4. Конфигурация

Формат конфигурации: **YAML**

Файл:

```
~/.config/app-powerd/config.yaml
```

---

# 5. Структура конфигурации

## version

Версия формата конфигурации.

```
version: 1
```

---

## defaults

Общие настройки сервиса.

```
defaults:
  enabled: true

  mode:
    ac: disable
    battery: enable

  timing:
    suspend_delay_ms: 30000
    resume_grace_ms: 3000
    min_suspend_ms: 5000

  maintenance_resume:
    enabled: false
    interval_ms: 30000
    duration_ms: 1000

  guards:
    audio_active: skip
    mic_active: skip
    camera_active: skip
    fullscreen: skip
```

---

## profiles

Повторно используемые политики.

Пример:

```
profiles:
  ignore:
    action: ignore

  freeze:
    action: freeze
    suspend_delay_ms: 60000

  throttle:
    action: throttle
    nice: 5
    cpu_weight: 20
    cpu_quota: "40%"
```

---

## rules

Правила сопоставления приложения с профилем.

Пример:

```
rules:
  - id: chrome
    match:
      executable: [google-chrome, chromium]
    policy:
      use_profile: throttle

  - id: telegram
    match:
      executable: [telegram-desktop]
    policy:
      use_profile: freeze
      suspend_delay_ms: 120000
```

---

# 6. Механизм сопоставления

Поддерживаемые поля match:

```
executable
cmdline_regex
wm_class
app_id
desktop_file
window_title_regex
```

# 7. Архитектура сервиса

Рекомендуемая модульная архитектура:

```
core
 ├─ config
 ├─ rules_engine
 ├─ state_machine

system
 ├─ process_inspector
 ├─ cgroup_manager
 ├─ freeze_backend
 ├─ throttle_backend

desktop
 ├─ focus_backend
 │   ├─ x11
 │   └─ wayland
 └─ window_tracker
```

---

# 8. Машина состояний приложения

Каждое приложение должно иметь состояние:

```
ACTIVE
BACKGROUND
THROTTLED
FROZEN
```

Переходы:

```
ACTIVE -> BACKGROUND
BACKGROUND -> THROTTLED
BACKGROUND -> FROZEN
FROZEN -> ACTIVE
THROTTLED -> ACTIVE
```

Сервис должен предоставлять CLI:

```

app-powerd freeze <pid>
app-powerd thaw <pid>
app-powerd reload-config
```

Сервис должен поддерживать запуск через:

```
systemd --user
```

```
~/.config/systemd/user/app-powerd.service
```

# 12. Метрики и диагностика

Сервис должен предоставлять:

```
app-powerd stats
```

Пример:

```
apps_frozen_total
apps_throttled_total
cpu_time_saved
wakeups_prevented
```

---

# 13. Требования к производительности

Сервис должен:

* потреблять менее 10MB RAM
* использовать менее 0.1% CPU
* не создавать заметных wakeups

# Итог

Результатом разработки должен стать **лёгкий user-level демон Linux**, который:

* автоматически управляет фоновыми приложениями
